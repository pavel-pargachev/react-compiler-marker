# React Compiler Marker

Highlights components optimized by the React Compiler with visual indicators directly in your editor.

## Features

- **Visual Markers**: See which components can be successfully optimized or fail to compile.
- **Inline Diagnostics**: Get detailed information about compilation results
- **Preview Output**: View the compiled code generated by React Compiler
- **Real-time Feedback**: Automatic updates as you edit your code

## Configuration

| Setting | Default | Description |
|---------|---------|-------------|
| `reactCompilerMarker.babelPluginPath` | `node_modules/babel-plugin-react-compiler` | Path to babel-plugin-react-compiler in your project |
| `reactCompilerMarker.configFile` | `.config/react-compiler.ts` | Path to a config module exporting `reactCompilerPaths` |

### React Compiler config file

Create a config file at `.config/react-compiler.ts` (or set `reactCompilerMarker.configFile` to another workspace-relative path).

Paths listed there get full React Compiler optimization; other source files use annotation mode (`'use memo'` opt-in).

Example (`.config/react-compiler.ts`):

```js
export const reactCompilerPaths = [
  "src/app",
  "src/features",
];
```

This list defines parts of the code that use `compilationMode: infer`, the rest of the code uses annotation mode (`'use memo'` opt-in).

## Requirements

Install `babel-plugin-react-compiler` in your project:

```bash
npm install babel-plugin-react-compiler
```

## Commands

- **Activate Extension**: Enable the marker
- **Deactivate Extension**: Disable the marker
- **Check in the current file**: Run compilation check on active file
- **Preview Compiled Output**: View the React Compiler's output code

---

### Indications

- **In source** — When a component has `'use memo'` in its body, that quoted text is highlighted in gray. Hover shows compiler status and actions.
- **Code Lens** — When the component has no `"use memo"` / `"use no memo"`, a **Code Lens** row shows `'use memo';` above the first body line.
- **Errors** — Red strikethrough on in-source `'use memo'`; virtual failures use a red `$(error)` Code Lens with strikethrough in the label.

Ensure **Code Lens** is enabled (`editor.codeLens`: true). Reload the window after updating (`Developer: Reload Window`).
